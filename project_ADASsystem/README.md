# 아두이노 기반 ADAS(전방 충돌방지 보조) HIL 시뮬레이터 프로젝트

> **개발 기간:** 2026.07.20 ~ 2026.07.21 
> **주요 기술 스택:** Arduino (C/C++), Sensors (HC-SR04, Joystick, 1602 LCD, Servo Motor), Tinkercad  

---

## 📌 1. 프로젝트 개요 및 동기

자율주행시스템공학과 전공생으로서 '배운 전공 지식을 아두이노 환경에서 직접 구현해볼 수는 없을까?'라는 고민에서 출발한 프로젝트입니다. 최종 목표인 **자율주행 RC카**를 제작하기에 앞서, 실제 자동차 산업에서 사용하는 **HIL(Hardware-in-the-Loop) 벤치 테스트** 기법에서 아이디어를 얻어, 책상 위에서 센서 제어 및 제어 로직을 먼저 검증하는 시뮬레이터를 개발하였습니다.

특히 **전공 과목(자율주행시스템개론)**에서 배운 **ADAS(첨단 운전자 지원 시스템)** 기술 중 **FCA(전방 충돌방지 보조 - Forward Collision-Avoidance Assist)** 기능을 핵심 로직으로 구현하였습니다. 단순 충돌 경고에 그치지 않고 위험 상황 시 운전자의 조향을 무시하고 안전 각도로 강제 개입하는 **조향 보조 회피 제어**를 시뮬레이션하였습니다.

추후 RC카 차체가 구비되면, 본 프로젝트에서 검증된 로직에 DC 모터 드라이버(L298N) 제어 코드만 추가하여 완벽한 자율주행 RC카로 확장할 예정입니다.

## 🛠️ 2. 시스템 아키텍처 및 구성 부품

| 구분 | 부품 / 기술 | 역할 및 기능 설명 |
| :--- | :--- | :--- |
| **인지 (Input)** | **초음파 센서 (HC-SR04)** | 전방 장애물과의 거리(cm) 실시간 측정 |
| **인지 (Input)** | **2축 조이스틱 모듈** | 운전자의 가상 조향(X축) 및 전진/후진(Y축) 명령 입력 |
| **판단 (Control)** | **Arduino UNO** | 거리 판단 알고리즘 처리 및 FCA 제어권 전환 |
| **제어 (Output)** | **서보모터 (Servo Motor)** | 조향 축(Steering Axis)을 담당하여 방향 제어 시뮬레이션 |
| **출력 (HMI)** | **1602 LCD (16핀)** | 주행 상태(Accel/Stop/Back), 장애물 거리, FCA 경고 문구 출력 |
| **출력 (HMI)** | **가변저항 (10KΩ)** | 1602 LCD의 밝기 대비(V0 핀 전압) 조절용 |
| **출력 (HMI)** | **경고 LED** | FCA 긴급 개입 발생 시 시각적 위험 경고 표시 |

## 💻 3. 핵심 제어 로직 (FCA 알고리즘)

* **정상 주행 모드 (Normal Mode):**
  * 조이스틱 X축 입력값을 서보모터 각도로 매핑하여 직관적인 수동 조향 수행.
  * 조이스틱 Y축 입력값에 따라 LCD 상단에 `Accel`, `Stop`, `Back` 상태 표시.
  * 초음파 센서로 감지된 실시간 거리를 LCD 하단에 표시 (잔상 방지 예외 처리 포함).

* **FCA 긴급 개입 모드 (Fail-Safe ADAS Mode):**
  * 전방 장애물 거리가 15cm 미만 감지 시 즉시 운전자 조작을 무시하고 **강제 회피 조향** 실행.
  * 경고 LED 점등 및 LCD 상에 경고 메시지 출력.

## 📝 5. 아쉬운점 및 개선 방향

* **하드웨어 배선 및 공간 효율성 (브레드보드 배선):**
  * 여러 센서와 디스플레이가 동시에 결합되면서 브레드보드 전원 레일 및 배선 꼬임 현상을 경험했습니다. 임베디드 시스템은 한정된 자원과 공간을 효율적으로 활용해야 하므로, 전원 버스 라인 구성 및 배선 정리의 중요성을 체감했습니다.
* **디버깅 및 프로그래밍 기초:**
  * 세미콜론 누락, 연산자 오타(`=` vs `==`) 등 사소한 문법적 실수가 예기치 못한 동작 에러로 이어지는 경험을 통해 단단한 C/C++ 기초 공부의 필요성을 깨달았습니다.
* **코드 구조화 및 모듈화 (SW 확장성):**
  * 현재는 `loop()` 구조 내에 조건문 위주로 코드가 작성되어 있으나, 추후 유지보수와 가독성을 위해 **사용자 정의 함수(Custom Functions) 분리** 및 **클래스(Class)를 활용한 센서 객체화** 기법을 도입하여 코드 효율성을 높일 예정입니다.
